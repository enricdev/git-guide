Baixa't git per a Windows

Baixa't git per a Linux

crea un nou repositori

crea un nou directori, obri'l i executa
git init
per a crear un nou repositori de git.

Fes checkout a un repositori

crea una copia d'un repositori executant l'ordre
git clone /path/to/repository
si utilitzes un servidor remot, has d'executar
git clone username@host:/path/to/repository

flux de treball

el teu repositori local està format per tres "arbres" gestionats per git. el primer és un Directori de treball que conté els fitxers, el segon és l'Index el qual actua com una àrea intermitja i finalment el HEAD que fa referència a l'últim commit que has executat.

add & commit

Pots proposar canvis (afegir-los a l'Index) utilitzant
git add <filename>
git add *
Aquest és el primer pas en un flux de treball bàsic de git. Per a consolidar aquests canvis has d'utilitzar
git commit -m "Commit message"
Ara els canvis al fitxer estan registrats al HEAD però encara no al repositori remot.

enviament dels canvis

Els canvis que has fet estan ara en el HEAD de la teua copia local. Per a enviar-los al teu repositori remot executa
git push origin master
Canvia master per la branca a la que vulgues enviar els canvis.

Si no has clonat un repositori que ja existeix i vols conectar el teu repositori a un servidor remot, has d'afegir-lo amb
git remote add origin <server>
Ara ja pots enviar els canvis al servidor remot.

vranques

Les branques s'utilitzen per desenvolupar funcionalitats aïllades les unes de les altres. La branca master és la branca per defecte quan crees un repositori. Utilitza altres branques durant el desenvolupament i barreja-les en la branca master quan hages acabat.

crea una nova branca anomenada "feature_x" i canvia't a ella utiltizant
git checkout -b feature_x
torna a master
git checkout master
i esborra la branca
git branch -d feature_x
una branca no està disponible per a tercers a no ser que la puges al repositori remot
git push origin <branch>

Actualitza i fusiona

per a actualitzar el teu repositori local amb els canvis més recents, executa
git pull
en el teu repositori local per a baixar i fusionar els canvis remots.
per a fusionar una altra branca amb la teua branca activa (p.ex. master), utilitza
git merge <branch>
en tots dos casos git intentarà fusionar automàticament els canvis. Malauradament, açò de vegades no és possible i es generen conflictes. Tu eres el responsable de fusionar aquests conflictes editant els fitxers que t'indica git. Després de modificar-los, has de marcar-los com a fusionats amb
git add <filename>
abans de fusionar els canvis pots revisar-los utilitzant
git diff <source_branch> <target_branch>

Etiquetes

és recomanable crear etiquetes per cada versió de programari. Este concepte no és nou ja que també existix en SVN. Pots crear una nova etiqueta amb el nom 1.0.0 executant
git tag 1.0.0 1b2e1d63ff
el 1b2e1d63ff fa referència als 10 primers caràcters del commit id al que has etiquetat. Pots obtindre la id del commit mirant el..

log

de la manera més simple, pots revisar l'historial del repositori amb l'ordre git log
Pots personalitzar afegir paràmetres al log per mostrar-lo com tu vulgues. Per veure només els commits d'un determinat autor:
git log --author=bob
Pots veure una versió comprimida del log on cada línia mostra un commit:
git log --pretty=oneline
També pots mostrar-lo en un arbre ASCII amb totes les branques, amb els noms de les etiquetes i les branques:
git log --graph --oneline --decorate --all
I veure només els fitxers que han canviat:
git log --name-status
Aquests són només alguns dels paràmetres que pots utilitzar. Per saber-ne més, consulta git log --help

Substitueix canvis locals

En cas que hages comés algun error, cosa que segur mai et passa ;), pots restablir els canvis locals utilitzant l'ordre
git checkout -- <filename>
Açò substitueix els canvis del teu directori de treball per l'últim contingut del HEAD. Els canvis que ja estiguen afegits a l'index, així com els nous fitxers, es mantindran.

Si en lloc d'això vols eliminar tots els teus canvis locals i commits, pots baixar l'última versió del servidor i apuntar a la teua branca master local
git fetch origin
git reset --hard origin/master
